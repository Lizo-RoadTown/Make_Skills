# Plans

Dated "what to do next" plan files, generated by the [`next-actions-planning`](../../skills/next-actions-planning/SKILL.md) skill.

## Why these are dated

A plan is a snapshot of priorities at a point in time. The repo, the conversation, and the user's signals all change. Re-running the planning skill writes a *new* file — it does not edit a prior one. The dated filename means:

- The next planning pass can read what was decided last time and pick up from there.
- Stale plans stay readable as historical context (why we did X then, what was on the table).
- No merge conflicts on a single living `PLAN.md`.

## Filename convention

`YYYY-MM-DD-<short-name>.md` — usually just `next-actions.md` for the standard pass.

## When to skip writing a new one

Per the skill, skip if a plan was written within the last 24 hours and nothing material has shipped since. Re-read the existing plan instead.

## Reading order

The most recent file is authoritative for what's-next. Older files are context for *how we got here*.
